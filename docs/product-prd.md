## 1. 产品概述

### 1.1 产品定位

**复盘神器**是一款跨平台（iOS + Android）的个人复盘工具。用户设定年度目标后，每天通过语音或文字自由输入经历和反思，AI 自动按 GDRR 框架（Goal-Result-Difference-Reason / 目标-结果-差异-根因）结构化整理，并结合年度目标以教练角色追问盲区、提供对齐目标的建议。

### 1.2 一句话描述

"设定目标，每天说几句，AI 帮你结构化反思，追踪你的成长轨迹。"

### 1.3 核心差异化

| 传统日记/复盘工具 | 复盘神器 |
|------------------|---------|
| 手动打字整理 | 语音/文字随意输入，AI 自动结构化 |
| 独立记录，无关联 | 每条复盘自动关联年度目标，追踪进度 |
| 被动记录 | AI 教练主动追问盲区，给出成长建议 |
| 用户配置 API Key | 后端托管，打开即用 |

## 2. 目标用户

### 2.1 核心用户画像

- **知识工作者 / 职场人**：有年度目标，需要定期反思和调整
- **自我成长践行者**：习惯记录但缺乏结构化方法
- **忙碌的个体**：需要极低摩擦的输入方式（语音 > 打字）

### 2.2 用户场景

| 场景 | 描述 | 输入方式 |
|------|------|---------|
| 每日复盘 | 下班后回顾今天，说说做了什么、遇到什么问题 | 语音为主 |
| 项目复盘 | 一个项目结束，总结成败得失 | 语音/文字 |
| 周度回顾 | AI 自动聚合一周的复盘，生成周报 | 自动生成 |
| 目标检视 | 查看年度目标进展，看看是否偏离方向 | 阅读报告 |
| 教练对话 | 针对某条复盘深入追问 | 文字对话 |

## 3. 功能模块

### 3.1 M0 — 目标设定

**描述**：用户首次进入 App 后，引导设定 1-3 个年度目标。

**功能要点**：
- 添加/编辑/删除年度目标
- 目标字段：标题、详细描述、类别（工作/人际/个人状态/个人生活）
- 关键结果（KR）：每个目标可设置多个可衡量的关键结果（jsonb）
- 进度追踪：用户手动调整 + AI 根据复盘内容建议更新
- 目标状态：进行中 / 已完成 / 已放弃

**优先级**：P0 — MVP 必需

### 3.2 M1 — 复盘输入

**描述**：核心交互入口。用户通过语音或文字输入复盘内容。

**功能要点**：
- **文字输入**：极简文本编辑区
- **语音输入**：调用腾讯云 ASR，实时转写 + 展示识别结果
- **Scope 选择器**：工作 / 人际 / 个人状态 / 个人生活 四象限
- **Typeless 式体验**：提交后 2-3 秒处理，自动纠错 + 结构化
- **提交后**：自动跳转详情页，展示 GDRR 结构化结果

**优先级**：P0 — MVP 必需

### 3.3 M2 — GDRR 结构化展示

**描述**：AI 将碎片的复盘文本按 GDRR 框架结构化，卡片式展示。

**GDRR 框架**：

| 组件 | 中文 | 说明 |
|------|------|------|
| Goal | 目标 | 当时想达成什么 |
| Result | 结果 | 实际发生了什么 |
| Difference | 差异 | 目标与结果的差距分析 |
| Reason | 根因 | 背后深层原因 |

**功能要点**：
- 四段卡片分层展示，每段独立可折叠
- 关联的年度目标标注
- AI 生成标签 chips
- 教练追问问题自动生成（2-3 个）
- 可选"深入聊聊"进入教练对话
- 编辑/删除操作

**优先级**：P0 — MVP 必需

### 3.4 M3 — AI 教练对话

**描述**：针对某条复盘，用户与 AI 教练进行多轮对话。教练结合用户画像记忆 + 年度目标 + 历史复盘，追问盲区、挑战假设、给出建议。

**功能要点**：
- 气泡式对话 UI
- 复盘提交后自动生成 2-3 个追问（首轮）
- 用户可点击追问快速回复，也可自由输入
- 多轮对话上下文关联（含用户画像 + 年度目标 + 近期洞察 + 历史模式）
- AI 回复以教练口吻：客观、建设性、略带挑战
- 对话历史持久化

**优先级**：P1 — Phase 2（MVP 只做首轮追问生成）

### 3.5 M4 — 首页

**描述**：进入 App 后的仪表盘。

**功能要点**：
- 今日复盘状态卡片
- 年度目标进度（横向滑动，显示完成百分比）
- 最近复盘列表（GDRR 摘要）
- 快捷入口：FAB 新建复盘
- 连续打卡天数

**优先级**：P0 — MVP 必需

### 3.6 M5 — 定时提醒

**描述**：本地推送通知，定时提醒用户复盘。

**功能要点**：
- 每日提醒时间设置（默认 21:00）
- 通知栏 Action：文字复盘 / 语音复盘 / 稍后提醒

**优先级**：P1 — Phase 2

### 3.7 M6 — 周期报告

**描述**：AI 定期自动生成复盘报告。

**报告类型**：日报/周报/月报/季报

**报告内容**：
- 复盘统计（数量、Scope 分布、标签 cloud）
- AI 叙事总结（教练口吻）
- 成长信号：技能观察 / 模式延续 / 关键突破
- 年度目标对齐度评估
- 下周期建议

**优先级**：P2 — Phase 3

### 3.8 M7 — 用户画像

**描述**：用户基础信息，注入 AI prompt 实现个性化。

**功能要点**：
- 昵称、职业/角色
- 当前关注领域
- 性格倾向（可选）
- 偏好工作时段

**优先级**：P1 — Phase 2

### 3.9 M8 — 历史列表

**描述**：所有历史复盘的列表视图，支持搜索和筛选。

**功能要点**：按时间倒序、Scope 筛选、关键词搜索、标签筛选

**优先级**：P1 — Phase 2

## 4. 用户核心流程

### 4.1 首次使用流程

```
下载 App → 手机号+验证码登录 → 设定年度目标（至少 1 个）
→ 填写基本画像（可跳过）→ 进入首页（空状态：引导第一次复盘）
```

### 4.2 每日复盘流程

```
打开 App → 输入页 → 选择 Scope → 录音/打字
→ 提交 → 2-3 秒 AI 处理 → 跳转详情页 → 查看 GDRR
→ 底部展示教练追问 → 可选：深入对话
```

## 5. 技术方案

### 5.1 技术栈

| 层 | 选择 |
|---|------|
| 前端 | React Native (Expo) + TypeScript |
| 后端 | Node.js + Express + TypeScript |
| 数据库 | PostgreSQL + Prisma ORM |
| 语音识别 | 腾讯云 ASR |
| AI 结构化 + 教练 | DeepSeek API |
| 短信验证码 | 腾讯云 SMS |
| 项目结构 | Monorepo（expo-app/ + server/ + shared/） |

### 5.2 API 设计

```
POST   /api/auth/send-code          ← 发送短信验证码
POST   /api/auth/verify-code         ← 验证码登录，返回 JWT

GET    /api/goals                    ← 目标列表
POST   /api/goals                    ← 创建目标
PUT    /api/goals/:id                ← 更新目标
DELETE /api/goals/:id                ← 删除目标

POST   /api/reviews/upload-audio     ← 上传录音，返回 ASR 识别文本
POST   /api/reviews                  ← 提交复盘文本，触发 AI 结构化
GET    /api/reviews                  ← 历史复盘列表（分页+筛选）
GET    /api/reviews/:id              ← 复盘详情（GDRR + 教练追问）
PUT    /api/reviews/:id              ← 修正复盘内容
DELETE /api/reviews/:id              ← 删除复盘

GET    /api/reviews/:id/coach-messages   ← 教练对话历史
POST   /api/reviews/:id/coach-messages   ← 发送消息，返回教练回复

GET    /api/user/profile             ← 用户画像
PUT    /api/user/profile             ← 更新画像
```

### 5.3 数据库核心表

- **User**：id, phone, nickname, profile_json, created_at
- **AnnualGoal**：id, user_id, title, description, category, key_results(jsonb), progress, status, created_at
- **Review**：id, user_id, raw_text, audio_url, scope_area, gdrr_goal, gdrr_result, gdrr_difference, gdrr_reason, tags(jsonb), coach_questions(jsonb), insight_candidates(jsonb), growth_signals(jsonb), related_goals(jsonb), created_at
- **CoachMessage**：id, review_id, role(user/coach), content, created_at
- **UserInsight**：id, user_id, category(blind_spot/strength/pattern/skill), insight, confidence, created_at
- **Report**：id, user_id, type(week/month/quarter), period_start, period_end, content_json(jsonb), created_at

## 6. 视觉设计

### 6.1 设计方向

**暖色纸质日记风** — 模拟纸质笔记本的温暖感觉，让用户有"翻开本子写日记"的熟悉感。

### 6.2 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | `#FFF8F0` | 暖白/米色底，模拟纸张 |
| 卡片背景 | `#FFFBF7` | 略深的纸质卡片 |
| 主色调 | `#C4724B` | 暖棕，像是旧书的封面色 |
| 强调色 | `#D4895E` | 稍亮的棕，hover/active 状态 |
| 文字主色 | `#3C2E26` | 深棕黑，像墨水色 |
| 文字辅助 | `#8B7355` | 浅棕，次要文字 |
| 成功/积极 | `#6B9F6E` | 墨绿，不刺眼 |
| 警示/关注 | `#D4956B` | 暖金棕 |
| 分割线 | `#E8D5C4` | 浅米色 |

### 6.3 设计原则

- **像纸一样简洁**：少用卡片阴影，多用空白和层次来区分内容
- **克制动画**：过渡用 fade，不要弹跳，保持安静的氛围
- **字形优先**：标题用衬线体唤起阅读感，正文用无衬线保证可读性
- **微妙的纹理**：背景可选加入极淡的纸质纹理（paper texture），增强触感联想

### 6.4 参考意象

- 一本旧皮面笔记本翻开的感觉
- MUJI 式简约
- 深夜台灯下写字的那种安静专注感
